### 评价指标描述

#### 目标函数（Objective Function）

| **序号** | **指标名称** | **描述** |
| :--- | :--- | :--- |
| **1** | **长期平均系统成本 (Long-Term Average Cost)** | $$\bar{C} = \lim_{T\to\infty}\frac{1}{T}\sum_{t=1}^{T}\text{Cost}(t)$$ <br> (包含能耗、云服务费与切换惩罚的加权和) |
| **选择原因** | 这是 OLMA 算法的**核心优化目标**。它综合反映了系统在长期运行中的经济效益和综合能效。**最小化此指标**是算法设计的最主要目的。 | |

-----

#### 关键评价指标 (Key Performance Indicators, KPIs)

| **序号** | **指标名称** | **逻辑层次** | **描述** |
| :--- | :--- | :--- | :--- |
| **2** | **平均端到端时延 (Average End-to-End Delay)** | **服务质量 (QoS)** | 统计任务从产生到完成（含排队、传输、计算及回传）的总时长。 |
| **选择原因** | 它是衡量 VEC 系统**服务质量 (QoS)** 的核心指标。用于验证算法在追求低成本的同时，是否牺牲了用户的等待时间，特别是对于时延敏感型任务。 | | |
| **3** | **单时隙决策时延 (Decision Time per Slot)** | **落地可行性** | 算法在每个时间槽内计算出联合决策（匹配+资源分配）所需的 CPU 执行时间。 |
| **选择原因** | 鉴于 VEC 的高动态性（时隙通常仅 100ms），算法必须具备**毫秒级**的决策速度才能满足**实时控制**要求。这是衡量算法能否在实际工程中落地的关键约束。 | | |
| **4** | **单任务能耗 (Energy Consumption per Task)** | **绿色计算** | 平均处理每个任务单位所需的能量消耗（单位：焦耳/任务）。 |
| **选择原因** | 该指标独立于总成本，专注于**能源效率**。它用于验证 OLMA 算法是否通过精细的连续资源分配（如功率控制、频率调整）有效降低了不必要的能源浪费，体现绿色计算优势。 | | |
